# 更新日志

## v3.0.0 — 2026-05-03

### 新增（7 个算法方向）
- **搜索树可视化**：MCTS `get_search_tree()` + `/api/search-tree` 端点 + 前端 SVG 树图
- **牌局回放**：`/api/replay` 端点，支持步进导航查看历史牌局
- **遗传算法优化**（`src/analysis/genetic.py`）：种群进化、适应度评估、交叉变异、精英保留
- **模式挖掘**（`src/analysis/pattern.py`）：致胜开局分析、死局前兆检测、胜败对比
- **策略热切换**：`/api/switch-strategy` 端点，运行时切换策略
- **策略对抗锦标赛**（`src/analysis/tournament.py`）：同 seed 两两对比、胜率矩阵、排名
- **神经网络评估**（`src/strategy/neural.py`）：
  - `SimpleMLP`：纯 numpy 实现的多层感知器（input→128→64→1）
  - `NeuralTrainer`：数据收集 + 反向传播训练 + 模型保存/加载
  - `NeuralStrategy`：实现 Strategy Protocol，用 MLP 评估移动
  - 58 维特征提取（每列 5 维 × 10 列 + 8 维全局）
- **注册中心新增 `neural` 策略**

### API 新增
| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/search-tree` | GET | MCTS 搜索树数据 |
| `/api/replay` | GET | 牌局回放数据 |
| `/api/switch-strategy` | POST | 运行时切换策略 |
| `/api/genetic-optimize` | POST | 遗传算法参数优化 |
| `/api/mine-patterns` | POST | 模式挖掘分析 |
| `/api/tournament` | POST | 策略对抗锦标赛 |
| `/api/neural-train` | POST | 神经网络训练 |

### 变更
- 版本号 v2.0.0 → v3.0.0

## v2.0.0 — 2026-05-03

### 新增
- **策略注册中心**（`src/strategy/registry.py`）：单一注册点，消除 4 处重复注册
  - `get_strategy(name)` / `list_strategies()` / `register_custom()`
  - 插件扩展点：`register_custom()` 支持运行时注册新策略
- **多策略并行对比**（`src/analysis/compare.py`）
  - `ParallelStrategyRunner`：同 seed 多策略运行，公平对比
  - `ComparisonReport`：排名、对比矩阵、每局详情
  - API: `/api/compare`
- **策略弱点自动检测**（`src/analysis/weakness.py`）
  - 8 维因子阈值检测（高/中/低严重度）
  - 自动生成改进建议
  - API: `/api/weakness`
- **标准化场景库**（`src/analysis/scenario.py`）
  - `ScenarioLibrary`：从大量游戏中提取决策点，构建可复现测试集
  - `ResponseCollector`：记录策略在标准场景上的决策响应
  - 响应矩阵：7 维特征（花色保持/翻牌/空列/发牌/序列/速度/位置）
- **因子分析器**（`src/analysis/factor.py`）
  - `FactorAnalyzer`：从响应矩阵提取 8 维策略因子
  - 因子得分对比表 + 解释方差
  - TXT 报告导出
- **策略参数自动调优**（`src/analysis/tuning.py`）
  - `ParameterTuner`：网格搜索最优参数组合
  - 支持自定义评分标准（胜率/完成/效率）
  - 基线对比 + 提升幅度

### 变更
- 版本号 v1.3.0 → v2.0.0
- `src/__init__.py` 版本号改为从 pyproject.toml 动态读取
- `pyproject.toml` 补全所有运行时依赖（psutil/GPUtil/waitress）
- `spiderette.spec` 补全 hiddenimports（waitress/psutil/GPUtil/profile/exporter）
- server.py 添加全局错误处理器（500/404/405 统一 JSON 返回）
- 所有策略构建改为通过注册中心（server/batch/runner/main 4 处统一）
- UI 策略量化面板：8 因子实时计算 + 进度条 + 计算方程

### 架构改进
- 策略注册中心：4 处重复 → 1 处注册，插件可扩展
- 多策略对比：串行运行 → 同 seed 并行对比
- 因子分析：静态模板 → 基于场景库的动态测量
- 参数调优：手动调参 → 网格搜索自动优化

## v1.1.0 — 2026-05-03

### 新增
- **原生窗口**：pywebview + WebView2 独立窗口，无需浏览器
- **生产级服务器**：Waitress WSGI 替代 Flask 开发服务器（8线程池，连接超时）
- **SSE 实时推送**：服务端主动推送游戏状态，零轮询开销
- **网络健壮性**：请求超时(5s)/重试(3次)/指数退避/请求去重/inflight 复用
- **自适应轮询**：根据服务器响应延迟动态调整频率（100ms-2000ms）
- **系统监控**：CPU/内存/GPU 实时监控（psutil + GPUtil），EMA 平滑
- **状态机控制**：新牌局/启动/暂停/停止 完整状态流转，按钮明暗联动
- **详细运行日志**：自动检测每步操作（移牌/发牌/完成序列），SSE 实时推送
- **牌局快照**：文本格式牌局状态，步进导航（第一步/上一步/下一步/最后一步）
- **UI 重构**：14 模块对齐 WorkBuddy 布局（策略监控/性能监控/牌局详情/快照/统计/日志/分析）
- **多进程并行**：批量模拟使用 ProcessPoolExecutor，支持 CPU 多核加速
- **基准测试框架**：BenchmarkRunner 标准化策略评估，多格式报告导出
- **PyInstaller 打包**：一键打包为独立 .exe（~54MB），零安装部署
- **项目开发规范**：14 章 + 3 附录，覆盖安全性/稳定性/兼容性/拓展性
- **版本号动态化**：从 pyproject.toml 读取，API `/api/version` 注入前端

### 变更
- Flask 开发服务器 → Waitress 生产级 WSGI 服务器
- 10ms 轮询 → SSE 推送 + 自适应兜底轮询
- `disabled` HTML 属性控制按钮状态（替代 CSS class）
- 策略监控名称与下拉框中文文本同步
- 性能监控刷新频率 2s → 1s，CPU/内存/GPU 使用 EMA 平滑

### 修复
- 点击新牌局后按钮灰色不可操作（HTML `disabled` 属性未移除）
- 网络错误导致 UI 崩溃（Waitress + 重试 + 限频 toast）
- 策略名称显示英文 key 而非中文

### 依赖
- 新增：`pywebview>=5.0`, `psutil>=5.9`, `GPUtil>=1.4`, `waitress>=3.0`
