## v5.0.4 (2026-05-07) — 自定义策略导入 + 防御性测试

### 新增
- `src/ui/routes/system.py` — 新增 `/api/import-strategy` 端点：接收 manifest JSON → 校验 → 注册为可用策略 → 返回更新后的策略列表
- `src/core/manifest.py` — `StrategyManifest` 新增 `base_strategy` 字段（指定基础策略，如 `mcts`、`greedy`）
- `src/ui/static/index.html` — 策略下拉框改为动态加载（从 `/api/strategies` 获取）；新增「导入策略」按钮 + 弹窗 UI（manifest JSON 编辑 + 文件上传）
- `manifests/custom_mcts_v2.json` — 示例自定义策略 manifest

### 修复
- 🔴 **P0-1** `registry.py` 自引用无限递归 — `register_from_manifest()` 用 `manifest.name` 同时当注册名和基础策略名，导致 `get_strategy()` → `factory()` → `get_strategy()` 死循环。修复：新增 `base_strategy` 字段，`register_from_config()` 增加自引用即时校验
- 🟡 **P1-1** `registry.py` 未知基础策略延迟报错 — `register_from_config()` 在注册时不校验 `base_strategy` 是否存在，直到运行时才抛 `RecursionError`。修复：注册时即时校验，提前抛出 `ValueError`
- 🟡 **P1-2** 前端策略列表硬编码 — HTML 中 `<select>` 写死 6 个 `<option>`，新增策略后前端不更新。修复：改为 JS 动态加载

### 测试
- 新增 `TestImportStrategyAPI`（5 项）：正常导入、缺 manifest、缺 base_strategy、自引用检测、导入后可用性
- 新增 `TestFrontendBackendConsistency`（3 项）：无硬编码选项、有导入按钮、调用正确端点
- 新增 `TestStrategiesAPI` 扩展（2 项）：所有内置策略精确匹配、API 与 registry 一致性
- 新增 `TestRegisterCustom` 扩展（2 项）：自引用报错、未知基础策略报错
- 总数从 291 增至 304

## v5.0.2 (2026-05-06) — 评分函数重构 + MCTS 搜索优化

### 修复
- 🔴 **P0-1** `heuristics.py` 状态哈希缺陷 — `GameState` 的 `__hash__` 未包含暗牌信息，导致 MCTS 节点缓存错误合并不同状态，决策失真
- 🟡 **P2-1** `heuristics.py` 评分函数合并 — `_score_same_suit_sequences` 与 `_score_column_order` 合并，消除同花色降序段的双重计分
- 🟡 **P2-2** `heuristics.py` 明牌评分线性化 — `_score_face_up` 从三角数计分改为线性计分（每张明牌 1 分），信息价值与数量成正比
- 🟡 **P2-3** `mcts.py` UCB1 归一化改进 — 从 `avg_score / 100` 硬编码改为 `tanh(avg_score / 50)` 压缩，不再依赖固定分数范围
- 🟡 **P2-4** `mcts.py` 缓存匹配全量化 — `_find_move_by_score` 从只评估前 5 个移动改为评估全部合法移动

### 优化
- 🟢 **P3-5** `heuristics.py` Rollout 温度退火 — `heuristic_rollout` 贪心概率从 0.5 线性增长到 0.95，前期多探索后期多利用
- 🟢 **P3-7** `mcts.py` 收敛检测动态化 — `_check_convergence` 根据 `assess_complexity` 动态调整阈值和最小访问次数，复杂局面不会过早停止
- 🟢 **P3-8** `determinization.py` 约束采样 — `sample_determinization` 有上方明牌的列优先分配且只选 rank < 上方明牌的候选牌，提高采样有效性

### 测试
- 268 项测试全部通过
