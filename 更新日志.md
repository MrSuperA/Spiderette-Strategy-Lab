## v5.0.0 (2026-05-04) — 策略迭代闭环 + Waitress 性能修复

### 策略迭代引擎（核心新增）
- `src/iteration/engine.py` — 完整迭代引擎：评估→分析→改进→对比的闭环
- `StrategyManifest` — 策略清单（可序列化/版本化/加载/比较，支持 `to_dict` + `from_dict`）
- `IterationRecord` — 迭代记录（基线评估→弱点分析→改进建议→改进后评估→前后对比）
- `ImprovementAction` — 可执行的改进建议（直接映射到策略参数名和建议值）
- `IterationEngine.iterate()` — 一键执行完整迭代，自动保存清单和记录到 `iterations/` 目录
- `IterationEngine.evolution_summary()` — 策略进化趋势摘要（含胜率趋势图）

### 分析模块反序列化修复（打通闭环）
- `genetic.py` — `GAResult` 新增 `strategy_name` 字段 + `from_dict()` 反序列化
- `tuning.py` — `TuningReport` / `TrialResult` 新增 `from_dict()`，`ParameterTuner` 新增 `apply_best(report)` 方法
- `weakness.py` — `WeaknessReport` / `Weakness` 新增 `from_dict()`，`THRESHOLDS` 新增 `param_hint` 映射，新增 `suggest_params()` 函数
- `registry.py` — 新增 `register_from_config()`（从参数字典注册策略变体）+ `register_from_manifest()`（从 JSON 文件加载策略）

### Waitress 队列积压修复
- Waitress 工作线程数 8→16
- 新增 `_task_executor`（4 线程后台任务池），重计算任务卸载到独立线程池
- 新增 `_submit_background()` / `_get_task_status()` 后台任务管理
- 新增 `/api/task/<task_id>` 端点 — 轮询后台任务状态
- 5 个重计算端点改为异步执行：`calc-factors`、`compare`、`tournament`、`genetic-optimize`、`iterate`

### 新增 API 端点（7 个）
- `POST /api/iterate` — 运行一次完整策略迭代
- `GET /api/iterations` — 获取迭代历史和进化摘要
- `POST /api/manifest/export` — 导出策略清单为 JSON
- `POST /api/manifest/load` — 从 JSON 加载策略清单并注册为可用策略
- `GET /api/manifests` — 列出所有已保存的策略清单
- `POST /api/weakness-suggest` — 弱点检测 + 可执行参数建议
- `GET /api/task/<task_id>` — 后台任务状态查询

### 文档自动同步增强
- `tools/sync_docs.py` — 新增 README.md 和更新日志.md 的自动更新
- README.md 策略表、架构图、统计数字全部从源码自动生成
- 更新日志.md 基于新旧 meta.json 差异自动追加变更条目（新增模块/策略/端点/测试数变化）
- `--check` 模式支持 CI 流水线文档一致性检查

### 测试

## v4.0.0 (2026-05-04) — 信息集搜索 + RL 框架

## v3.1.0 (2026-05-03) — 工程质量重构

### Bug 修复（8 个）
- 修复 `neural.py` 特征维度覆盖 bug — 移除错误的 58 维截断（B1）
- 修复 `neural.py` 评估逻辑错误 — 改为模拟执行后提取新状态特征（B2）
- 修复 `metrics.py` 置信区间上界计算错误（B3）
- 修复 `compose.py` LRU 缓存实际为 FIFO — 改用 OrderedDict（B4）
- 修复 `mcts.py` MCTSMemory 同为 FIFO（B5）
- 修复 `mcts.py` `_find_move_by_score` 空实现 — 改为模拟评估选最接近缓存分数的移动（B6）
- 修复 `heuristics.py` 重复函数 `_score_in_place_cards` — 合并到 `_score_same_suit_sequences`（B7）
- 修复 `compare.py` 异常处理创建不完整 GameResult — 补全所有必填字段（B8）

### 架构改进
- 新增 `src/core/exceptions.py` — 统一异常体系（SpideretteError → GameStateError / StrategyError / AnalysisError / ConfigError）
- 新增 `src/analysis/utils.py` — 公共工具函数（`run_single_game` / `run_games_batch` / `run_games_parallel`，消除 6 处游戏运行重复）

- `compose.py` Lambda 工厂改为具名函数（`GreedyStrategy()` / `RandomStrategy()`）
