# Spiderette Strategy Lab 项目开发规范

**文档版本**: v1.0.0
**最后更新**: 2026-05-03
**适用范围**: 所有 Spiderette Strategy Lab 项目开发与维护
**对应代码版本**: v1.0.0

## 📌 前言

本文档是 Spiderette Strategy Lab 项目的**强制性开发规范**，所有贡献者必须严格遵守。规范基于项目架构特性和历史经验教训（包括状态哈希缺陷、策略死循环、IntEnum 兼容性崩溃、多进程竞态等关键问题）制定，旨在确保代码质量、可维护性、安全性和可扩展性。

**违反本规范可能导致：**
- 策略决策错误（非法移动、死循环、循环检测失效）
- 统计数据失真（分布计算错误、置信区间偏差、除零崩溃）
- 多进程竞态条件（批量模拟数据污染、pickle 序列化失败）
- 状态不可逆（可变对象被意外修改，哈希链断裂）
- 打包后运行异常（路径/依赖缺失、WebView2 初始化失败）
- 未来扩展受阻（接口不统一、模块耦合过紧）

## 🏗️ 一、架构设计规范

### 1.1 四层解耦架构（核心原则）

**必须严格保持 4 层解耦，禁止跨层直接调用。**

```
┌─────────────────────────────────────────────────────────────┐
│ Layer 4: UI 层 (src/ui/)                                    │
│ 职责: Flask API、pywebview 窗口、前端交互、SSE 推送          │
│ 禁止: 包含策略逻辑或游戏规则判断                             │
│ ─────────────────────────────────────────────────────────── │
│ Layer 3: 分析层 (src/analysis/)                             │
│ 职责: 量化指标、基准测试、报告生成、批量模拟、实验运行       │
│ 禁止: 修改核心状态或策略逻辑                                 │
│ ─────────────────────────────────────────────────────────── │
│ Layer 2: 策略层 (src/strategy/)                             │
│ 职责: MCTS、贪心、随机等策略实现，启发式评估                 │
│ 禁止: 直接修改 GameState，只能通过 Move 表达意图             │
│ ─────────────────────────────────────────────────────────── │
│ Layer 1: 核心层 (src/core/ + src/envs/)                     │
│ 职责: 类型定义、规则引擎、状态管理、牌局生成、模拟环境       │
│ 禁止: 依赖上层模块（策略/UI/分析）                           │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 层间通信规范

**允许的依赖方向（只能向下依赖）：**
```
UI 层 ──→ 分析层 ──→ 策略层 ──→ 核心层
  │           │          │
  └───────────┴──────────┘  （同层模块可互相引用）
```

**允许的调用关系：**
- UI 层（server.py）→ 可以调用分析层（batch.py/runner.py）和策略层（构建策略实例）
- 分析层（runner.py/batch.py）→ 可以调用策略层（构建策略）和核心层（GameSession/SimulatorEnv）
- 策略层（mcts.py/compose.py）→ 只能调用核心层（Rules.legal_moves）

**禁止的调用关系：**
- ❌ 核心层（types/rules/session）导入策略层或 UI 层
- ❌ 策略层导入 UI 层或分析层
- ❌ 循环依赖（A → B → A）
- ❌ 通过 `import *` 隐式引入跨层依赖

### 1.3 接口设计规范

所有模块间通信必须通过 **Protocol 接口** 或 **冻结数据类**：

```python
# ✅ 正确：通过 Protocol 定义接口（src/core/types.py）
class Strategy(Protocol):
    @property
    def name(self) -> str: ...
    def choose(self, state: GameState, rules: Rules) -> Optional[Move]: ...

class Rules(Protocol):
    def legal_moves(self, state: GameState) -> list[Move]: ...
    def is_terminal(self, state: GameState) -> Outcome: ...
    def can_deal(self, state: GameState) -> bool: ...

class Environment(Protocol):
    def observe(self) -> GameState: ...
    def step(self, move: Move) -> None: ...
    def deal(self) -> None: ...
    def done(self) -> bool: ...
    @property
    def rules(self) -> Rules: ...

# ✅ 正确：通过 frozen dataclass 传递数据
@dataclass(frozen=True, slots=True)
class GameState:
    columns: tuple[Column, ...]
    stock: tuple[Card, ...]
    completed: int = 0
    difficulty: int = 2
    move_count: int = 0
    seed: int = 0

# ❌ 错误：使用可变对象传递状态
class GameState:
    def __init__(self):
        self.columns = []  # 可变，禁止
```

所有模块必须通过构造函数注入依赖，禁止硬编码实例化：

```python
class GameSession:

                 max_moves: int = 500, on_step: Optional[callable] = None):
        self.env = env
        self.strategy = strategy
        self.rules = env.rules
        self.max_moves = max_moves

# ❌ 错误：硬编码实例化
class GameSession:
    def __init__(self):
        self.env = SimulatorEnv()  # 硬编码
        self.strategy = GreedyStrategy()  # 缺少注入
```

## 🔒 三、安全性规范

### 3.1 状态不可变性（核心安全原则）

**GameState、Card、Column、Move 必须使用 `frozen=True, slots=True`，禁止运行时修改。**

```python
# ✅ 正确：创建新状态代替修改
new_state = replace(old_state, completed=old_state.completed + 1)

# ❌ 错误：直接修改冻结对象（会抛 FrozenInstanceError）
state.completed += 1
```

**理由：**
- 状态不可变性是循环检测（位置哈希）的基础
- MCTS 搜索树需要状态快照的可靠性
- 多进程并行需要状态的可序列化性
- StepRecord 保存的历史状态必须不可变

### 3.2 哈希安全性

状态哈希必须只包含**棋盘位置信息**，排除动态字段：

```python
# ✅ 正确：位置哈希（用于循环检测）
pos_hash = hash((state.columns, state.stock, state.completed))

# ❌ 错误：包含步数（每步都变，循环检测永远不触发）
pos_hash = hash(state)  # state.move_count 每步递增
```

**哈希用途清单：**
| 用途 | 哈希内容 | 位置 |
|------|---------|------|
| 循环检测 | `(columns, stock, completed)` | `session.py` |
| MCTS 状态缓存 | `(columns, stock, completed)` | `mcts.py` |
| 实验可复现 | `seed` | `SimulatorEnv(seed=...)` |

### 3.3 输入验证

所有外部输入（API 请求、配置文件、命令行参数）必须验证：

```python
# ✅ 正确：API 参数验证
@app.route("/api/new-game", methods=["POST"])
def new_game():
    data = request.get_json(silent=True) or {}
    seed = data.get("seed", 0)
    difficulty = data.get("difficulty", 2)
    if difficulty not in (1, 2, 4):
        return jsonify({"ok": False, "error": "难度必须为 1/2/4"}), 400
    if not isinstance(seed, int):
        return jsonify({"ok": False, "error": "seed 必须为整数"}), 400

# ❌ 错误：直接使用未验证的输入
difficulty = data["difficulty"]  # KeyError 或任意值
```

### 3.4 多进程安全

批量模拟使用 `ProcessPoolExecutor`，必须确保：

```python
# ✅ 正确：子进程函数可 pickle，独立构建策略
def run_single_game(args: tuple[int, int, str, int]) -> dict:
    seed, difficulty, strategy_name, max_moves = args
    strategy = _build_strategy(strategy_name)  # 子进程内独立构建
    env = SimulatorEnv(seed=seed, difficulty=difficulty)
    session = GameSession(env, strategy, max_moves=max_moves)
    result = session.run()
    return {"seed": seed, ...}  # 返回值传递

# ❌ 错误：共享不可序列化对象
def run_single_game(strategy, env):  # strategy 不可 pickle
    ...
```

**多进程约束清单：**
- 子进程函数必须在模块顶层定义（可 pickle）
- 不传递 lambda/闭包/不可序列化对象
- 每个子进程独立构建策略实例
- 结果通过返回值传递，不使用共享内存
- Windows 必须有 `if __name__ == "__main__"` 保护

### 3.5 资源限制

所有长时间运行的操作必须有上限：

| 资源 | 限制 | 实现位置 |
|------|------|---------|
| 单局最大步数 | 500 | `GameSession(max_moves=500)` |
| 批量模拟上限 | 500 局 | `min(data.get("count", 10), 500)` |
| MCTS 迭代上限 | 2000 | `create_mcts_deep(iterations=2000)` |
| MCTS 时间上限 | 2s | `time_limit=2.0` |
| SSE 队列大小 | 100 | `queue.Queue(maxsize=100)` |
| 循环检测窗口 | 20 步 | `CYCLE_WINDOW = 20` |
| 循环恢复阈值 | 3 次 | `cycle_strikes >= 3` |
| Flask 请求超时 | 30s | `threaded=True` |

## 🛡️ 四、稳定性规范

### 4.1 错误处理原则

**所有对外接口（API/CLI/窗口）必须捕获异常，不允许未处理异常导致进程退出：**

```python
# ✅ 正确：API 端点
@app.route("/api/step", methods=["POST"])
def step():
    try:
        # ... 业务逻辑
        return jsonify({"ok": True, ...})
    except Exception as e:
        return jsonify({"ok": False, "error": str(e)}), 500

# ✅ 正确：CLI 入口
def run_cli(...):
    try:
        session = GameSession(env, strategy, max_moves=500)
        result = session.run()
    except KeyboardInterrupt:
        print("\n[已中断]")
    except Exception as e:
        print(f"[错误] {e}")

# ✅ 正确：窗口入口
def run_window(...):
    try:
        _run_window(host=host, port=port)
    except SystemExit:
        pass
    except Exception as e:
        print(f"[错误] {e}")
        traceback.print_exc()
        input("\n按回车键退出...")
```

### 4.2 循环检测机制（核心稳定性保障）

GameSession 必须内置循环检测，防止策略陷入无限循环：

```python
# 循环检测参数
CYCLE_WINDOW = 20      # 检测窗口大小
CYCLE_THRESHOLD = 3    # 连续重复次数触发恢复

# 检测方式：位置哈希（不含 move_count）
pos_hash = hash((state.columns, state.stock, state.completed))
if pos_hash in recent_hashes:
    cycle_strikes += 1
else:
    cycle_strikes = 0

# 恢复策略：优先发牌 → 排除最近移动 → 随机替代
if cycle_strikes >= 3:
    if self.rules.can_deal(state):
        move = None  # 强制发牌
        cycle_strikes = 0
    elif legal_moves:
        alt_moves = [m for m in legal_moves if m not in recent_moves[-6:]]
        if not alt_moves:
            alt_moves = list(legal_moves)
        move = random.choice(alt_moves)
        cycle_strikes = 0
```

### 4.3 终态判断

策略实现必须在开始时检查终态：

```python
# ✅ 正确：策略入口检查终态
def choose(self, state: GameState, rules: Rules) -> Optional[Move]:
    if rules.is_terminal(state):
        return None
    moves = rules.legal_moves(state)
    if not moves:
        return None
    # ... 决策逻辑

# ❌ 错误：不检查终态
def choose(self, state, rules):
    moves = rules.legal_moves(state)  # 终态返回空列表
    return moves[0]  # IndexError
```

### 4.4 统计数据完整性

统计计算必须处理边界情况：

```python
# ✅ 正确：安全的统计计算
@property
def win_rate(self) -> float:
    return self.wins / self.total_games if self.total_games > 0 else 0.0

@property
def win_rate_ci95(self) -> tuple[float, float]:
    if self.total_games == 0:
        return (0.0, 0.0)
    # Wilson score interval ...

@property
def avg_moves(self) -> float:
    return self.total_moves / self.total_games if self.total_games > 0 else 0.0

# ❌ 错误：未处理除零
@property
def win_rate(self) -> float:
    return self.wins / self.total_games  # ZeroDivisionError
```

### 4.5 确定性与可复现性

所有实验必须支持种子复现：

```python
# ✅ 正确：种子传入环境
env = SimulatorEnv(seed=42, difficulty=1)
session = GameSession(env, strategy, max_moves=500)
result = session.run()  # 同一 seed 永远产生同一牌局

# ❌ 错误：使用随机种子但未记录
env = SimulatorEnv(seed=random.randint(0, 99999))  # 不可复现
```

**可复现性要求：**
- 实验配置必须保存 seed 列表
- 报告必须记录 seed 范围

## 🔧 五、兼容性规范

### 5.1 Python 版本兼容

- **推荐版本**: Python 3.12
- **禁止使用**: 3.13+ 独有特性

### 5.2 IntEnum 兼容性（Python 3.12 关键变更）

Python 3.12 中，IntEnum 类内部的非枚举成员会被视为枚举值：

```python
# ✅ 正确：辅助数据在模块级
_RANK_LABELS = {1: "A", 2: "2", ..., 13: "K"}

class Rank(IntEnum):
    ACE = 1
    TWO = 2
    # ...

# ❌ 错误：dict 放在 IntEnum 类内（Python 3.12 崩溃）
class Rank(IntEnum):
    _LABELS = {1: "A"}  # 被视为枚举成员！
    ACE = 1
```

### 违规案例 3：策略不检查终态

```python

    return rules.legal_moves(state)[0]  # IndexError

# ✅ 正确：先检查终态
def choose(self, state, rules):

        return None
    moves = rules.legal_moves(state)
    return moves[0] if moves else None
```

## 附录 A：术语表

| 术语 | 含义 |
|------|------|
| GameState | 牌局完整状态（不可变 frozen dataclass） |
| Move | 一次移牌操作（src_col + src_start + dst_col，dst_col=-1 表示发牌） |
| StepRecord | 单步执行快照（状态 + 移动 + 耗时 + 合法移动数） |
| GameResult | 一局完整结果（outcome + steps + total_moves + completed） |
| Strategy | 策略接口（Protocol：name + choose） |
| Rules | 规则接口（Protocol：legal_moves + is_terminal + can_deal） |
| Environment | 环境接口（Protocol：observe + step + deal + done） |
| Outcome | 游戏结果枚举（PLAYING / WIN / DEADLOCK） |
| MCTS | 蒙特卡洛树搜索（Monte Carlo Tree Search） |
| CI95 | 95% 置信区间（Wilson score interval） |
| DistributionStats | 数值分布统计摘要（均值/标准差/中位数/百分位） |
| GameMetrics | 单局详细量化指标 |
| StrategyStats | 单策略聚合统计 |
| BenchmarkRunner | 基准测试框架 |
| ProcessPoolExecutor | Python 多进程并行执行器 |
| pywebview | 原生窗口库（使用系统 WebView 引擎） |
| WebView2 | Microsoft Edge WebView2 运行时 |
