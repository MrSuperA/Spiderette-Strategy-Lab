# Spiderette Strategy Lab

**蜘蛛纸牌策略研究平台** — 纯算法驱动、仿真驱动的策略探索与评估框架

---

## 项目简介

Spiderette Strategy Lab 是一个专注于蜘蛛纸牌移牌策略研究的平台。通过系统化的模拟实验和量化分析，探索并对比不同策略在多难度下的表现。

**核心特点：**
- 🎯 **纯仿真驱动** — 无需游戏进程，内置牌局生成器和模拟环境
- 🧠 **MCTS 策略引擎** — 蒙特卡洛树搜索 + 贪心/随机基线策略
- 📊 **完整量化体系** — 胜率/置信区间/分布统计/效率指标/基准测试
- 🖥️ **独立原生窗口** — pywebview + WebView2，双击 .exe 即用
- ⚡ **多进程并行** — 批量模拟支持 CPU 多核加速
- 🔒 **生产级服务器** — Waitress WSGI，8 线程池，SSE 实时推送
- 📦 **一键打包** — PyInstaller 打包为独立 .exe，零安装部署

## 稳定性保障

| 机制 | 说明 |
|------|------|
| Waitress WSGI | 8 线程池，连接超时 30s |
| SSE 实时推送 | 服务端主动推送，零轮询开销 |
| 请求重试 | 3 次重试 + 指数退避 |
| 请求超时 | 5 秒 AbortController 超时 |
| 请求去重 | inflight Map 复用 |
| 自适应轮询 | 响应 <50ms 加速，>200ms 减速 |
| SSE 重连 | 指数退避（1s → 30s） |
| Toast 限频 | 网络错误每 3 秒最多一次 |

## 策略量化

每个策略通过 8 维决策因子 + 搜索行为 + 表现指标进行量化描述，支持多策略对比矩阵。

### 决策因子（8 维）

| 因子 | 标识 | 低分端 | 高分端 | 描述 |
|------|------|--------|--------|------|
| 花色保持倾向 | `suit_preservation` | 花色无关 | 严格同花色 | 选择保持花色一致的移动的倾向 |
| 翻牌意愿 | `exposure_willingness` | 保守 | 激进 | 选择翻开暗牌的移动的倾向 |
| 序列构建倾向 | `sequence_building` | 短视 | 长远 | 优先构建长序列而非短期收益 |
| 空列利用倾向 | `empty_column_usage` | 低频 | 高频 | 使用空列进行重组的频率 |
| 发牌时机偏好 | `deal_timing` | 延迟发牌 | 尽早发牌 | 在还有合法移动时选择发牌的倾向 |
| 可逆性偏好 | `reversibility_preference` | 不关心 | 优先可逆 | 选择可逆移动的倾向 |
| 风险容忍度 | `risk_tolerance` | 保守 | 冒险 | 选择可能破坏现有结构但高收益的移动 |
| 决策一致性 | `decision_consistency` | 随机 | 确定性 | 面对相似局面时做出相同选择的程度 |

每个因子包含：得分（0-1）+ 95% 置信区间 + 解释方差比例

### 搜索行为特征

| 指标 | 描述 |
|------|------|
| `avg_iterations` | 平均迭代次数 |
| `avg_depth` | 平均搜索深度 |
| `avg_branching` | 平均分支因子 |
| `avg_time_ms` | 平均搜索耗时 |
| `exploration_weight` | UCT 探索权重 |
| `exploitation_ratio` | 利用/探索比 |
| `cache_hit_rate` | 状态缓存命中率 |

### 表现指标（按难度分组）

| 指标 | 描述 |
|------|------|
| `win_rate` + 95% CI | 胜率及置信区间 |
| `moves` 分布 | 步数的均值/标准差/中位数/P25/P75/P90 |
| `completed` 分布 | 完成序列数的分布 |
| `efficiency` 分布 | 移牌效率（完成数/总步数）的分布 |
| `deal_ratio` 分布 | 发牌占比的分布 |
| `max_win_streak` | 最长连胜 |
| `max_lose_streak` | 最长连败 |
