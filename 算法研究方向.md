## 1. 神经网络评估

### 算法原理

传统 MCTS 使用随机 rollout 或简单启发式评估叶节点。神经网络评估替代这个过程：
- **策略头** (Policy Head): 预测每个合法移动的概率分布
- **价值头** (Value Head): 预测当前状态的胜率 [-1, 1]

```
输入: GameState 特征向量 (63维)
  ↓
特征提取器 (全连接层 / CNN)
  ↓
├── 策略头 → softmax → P(a|s)  每个动作的概率
└── 价值头 → tanh → V(s)      状态价值 [-1, 1]
```

### 特征工程

```python
# 当前已有 get_features() 返回 63 维向量
# 每列 5 维: [顶牌rank, face_up, 序列长度, 列长度, 花色一致性]
# 全局 3 维: [stock_remaining, completed, movable_count]
# 空列 1 维: [empty_cols]

# 建议扩展到 ~100 维:
# + 每列的花色分布 (4维 one-hot × 10列 = 40维)
# + 最近 5 步的移动历史 (5 × 3 = 15维)
# + 当前策略的因子得分 (8维)
```

### 实现路径

```
阶段 1: 数据收集
  用现有策略（greedy/mcts）玩 10000 局
  记录每个决策点的 (state, chosen_move, final_outcome)
  存储为训练数据集

阶段 2: 模型训练
  PyTorch MLP: 100 → 256 → 128 → 64 → (policy_head, value_head)
  损失函数: L = L_policy + L_value
    L_policy = CrossEntropy(预测概率, 实际选择)
    L_value  = MSE(预测胜率, 实际结果)
  训练: Adam, lr=1e-3, batch=256, 50 epochs

阶段 3: 集成到 MCTS
  在 MCTS 的叶节点评估中使用神经网络:
  def evaluate_leaf(state):
      policy, value = neural_net.predict(state)
      return value  # 替代随机 rollout

阶段 4: AlphaZero 式自弈
  策略通过自弈生成训练数据 → 训练 → 更强策略 → 更好的数据
  迭代 10-50 轮，每轮 1000 局自弈
```

### 接入方式

```python
class NeuralStrategy:
    @property
    def name(self) -> str:
        return "neural"

    def __init__(self, model_path: str):
        self.model = load_model(model_path)

    def choose(self, state: GameState, rules: Rules) -> Optional[Move]:
        if rules.is_terminal(state):
            return None
        features = extract_features(state)
        policy, value = self.model.predict(features)
        legal = rules.legal_moves(state)
        # 按 policy 概率选择
        probs = [policy[move_to_index(m)] for m in legal]
        return random.choices(legal, weights=probs, k=1)[0]
```

### 预期收益

- 胜率提升: 1花色 0% → 15-30%, 2花色 0% → 5-15%
- 决策质量: 从"贪心局部最优"到"全局价值评估"
- 可解释性: 策略头概率分布可视化

## 2. 强化学习（Q-Learning）

### 算法原理

Q-Learning 学习一个 Q(s,a) 函数，表示在状态 s 执行动作 a 的长期回报。

```
Q(s,a) ← Q(s,a) + α[r + γ·max Q(s',a') - Q(s,a)]
                     ↑     ↑              ↑
                   学习率  折扣因子      下一状态最大Q值
```

### 状态-动作空间分析

```
状态空间: ~10^120 (理论) → 实际可达 ~10^30 (1花色)
动作空间: 每步 0-20 个合法移动

挑战: 状态空间太大，无法用表格 Q-Learning
方案: Deep Q-Network (DQN)
```

### 实现路径

```
阶段 1: 离散化 Q 表（小规模验证）
  1花色模式，状态哈希为 key
  Q[hash(state), action_index] = value
  ε-greedy 探索 (ε=0.1 → 0.01 衰减)

阶段 2: DQN
  Q_network: state_features → [Q(s,a1), Q(s,a2), ...]
  Target Network: 稳定训练
  Experience Replay: 存储 (s, a, r, s') 转移
  训练: 从 replay buffer 采样 batch, 最小化 TD error

阶段 3: Double DQN + Dueling
  减少 Q 值过估计
  分离状态价值和动作优势
```

### 接入方式

```python
class DQNStrategy:
    @property
    def name(self) -> str:
        return "dqn"

    def choose(self, state: GameState, rules: Rules) -> Optional[Move]:
        legal = rules.legal_moves(state)
        if not legal:
            return None
        features = extract_features(state)
        q_values = self.model.predict(features)
        # 选择 Q 值最高的合法动作
        legal_q = [(m, q_values[move_index(m)]) for m in legal]
        return max(legal_q, key=lambda x: x[1])[0]
```

### 预期收益

- 自动发现策略模式，无需人工设计启发式
- 适应性：不同难度自动调整策略
- 训练成本：GPU 2-4 小时（1花色），8-16 小时（2花色）

## 3. 遗传算法优化

### 算法原理

将策略参数编码为"基因"，通过选择、交叉、变异进化出最优参数。

```
种群: [个体1(参数集), 个体2, ..., 个体N]
适应度: 每个个体运行 K 局，胜率 = 适应度
选择: 轮盘赌 / 锦标赛选择
交叉: 参数向量的均匀交叉
变异: 高斯噪声扰动
```

### 当前可优化参数

```toml

[[params]]
name = "iterations"
range = [50, 100, 200, 500, 1000]

[[params]]

[[params]]
name = "exploration_weight"
range = [0.5, 0.8, 1.0, 1.41, 2.0]

# 贪心策略参数（启发式权重）
[[params]]
name = "suit_weight"
range = [0.0, 0.5, 1.0, 2.0]

[[params]]
name = "expose_weight"
range = [0.0, 0.5, 1.0, 2.0]

[[params]]

range = [0.0, 0.5, 1.0, 2.0]
```

### 实现路径

```python
class GeneticOptimizer:
    def __init__(self, strategy_name, param_space, pop_size=20, generations=50):
        self.pop_size = pop_size
        self.generations = generations
        self.param_space = param_space

    def evolve(self, difficulty=1, games_per_eval=20):
        population = self._init_population()
        for gen in range(self.generations):
            # 评估适应度
            fitness = [self._evaluate(ind, difficulty, games_per_eval) for ind in population]
            # 选择
            parents = self._select(population, fitness)
            # 交叉 + 变异
            offspring = self._crossover_mutate(parents)
            # 替换
            population = self._survivor_select(population, offspring, fitness)
        return self._best(population)
```

### 预期收益

- 自动发现人工难以找到的参数组合
- 可并行评估（利用 `ProcessPoolExecutor`）
- 10-20 代进化即可看到明显提升

## 4. 多难度自适应

### 算法原理

不同难度（1/2/4花色）的游戏特征差异巨大：
- 1花色: 状态空间小，序列构建容易，贪心即可
- 2花色: 花色一致性成为关键因素
- 4花色: 需要长期规划，发牌时机极为重要

### 实现路径

```python
class AdaptiveEnvironment:
    """自适应环境：根据难度调整规则参数"""

    def __init__(self, base_difficulty: int):
        self.difficulty = base_difficulty
        # 不同难度的规则微调
        self.params = {
            1: {"deal_penalty": 0.0, "suit_bonus": 0.0},
            2: {"deal_penalty": 0.5, "suit_bonus": 1.0},
            4: {"deal_penalty": 1.0, "suit_bonus": 2.0},
        }[base_difficulty]

    @property
    def rules(self) -> Rules:
        return AdaptiveRules(self.params)
```

### 接入方式

实现 `Environment` Protocol 即可：
```python
class AdaptiveEnv:
    def observe(self) -> GameState: ...
    def step(self, move: Move) -> None: ...
    def deal(self) -> None: ...
    def done(self) -> bool: ...
    @property
    def rules(self) -> Rules: ...
```

### 当前状态

### 实现路径

```python
# MCTS 策略中添加搜索树导出
class MCTSStrategy:
    def get_search_tree(self) -> Optional[dict]:
        if not self._root:
            return None
        return self._node_to_dict(self._root, max_depth=5)

    def _node_to_dict(self, node, max_depth):
        if max_depth <= 0:
            return {"visits": node.visits, "truncated": True}
        return {
            "visits": node.visits,
            "value": round(node.value, 3),
            "move": str(node.move) if node.move else "root",
            "children": [
                self._node_to_dict(c, max_depth - 1)
                for c in sorted(node.children, key=lambda c: -c.visits)[:5]
            ]
        }
```

### 前端可视化

```
使用 D3.js 树图:
  节点大小 = log2(visits)
  节点颜色 = value (红→绿)
  边粗细 = visits 占比
  最佳路径高亮
```

## 9. 策略对抗

### 当前状态

`BenchmarkRunner` 支持多策略对比，但不是真正的"对抗"。

### 增强方案

策略对抗的核心思想：一个策略的弱点被另一个策略利用。

```python
class StrategyTournament:
    """策略锦标赛：每对策略在同一组种子上对比"""

    def run(self, strategies, seeds, difficulty):
        results = {}
        for i, s1 in enumerate(strategies):
            for j, s2 in enumerate(strategies):
                if i >= j:
                    continue
                # 同 seed 对比
                wins_s1 = 0
                for seed in seeds:
                    r1 = self._run_single(s1, seed, difficulty)
                    r2 = self._run_single(s2, seed, difficulty)
                    if r1.completed > r2.completed:
                        wins_s1 += 1
                results[(s1.name, s2.name)] = wins_s1 / len(seeds)
        return results
```

## 10. 模式挖掘

### 算法原理

从大量牌局数据中发现"致胜模式"：
- 什么样的开局配置更容易赢？
- 哪些移动序列经常出现在胜利局中？
- 死局前有什么共同特征？

### 实现路径

```python
class PatternMiner:
    """从历史数据中挖掘致胜模式"""

    def mine_winning_patterns(self, games: list[GameExportData]):
        wins = [g for g in games if g.outcome == "win"]
        losses = [g for g in games if g.outcome == "deadlock"]

        # 1. 开局特征对比
        win_opening = self._extract_opening_features(wins)
        loss_opening = self._extract_opening_features(losses)

        # 2. 关键移动序列
        win_sequences = self._extract_move_sequences(wins)
        common_win = self._find_common_subsequences(win_sequences)

        # 3. 死局前兆
        deadlock_signals = self._extract_deadlock_signals(losses)

        return {
            "winning_openings": win_opening,
            "common_sequences": common_win,
            "deadlock_signals": deadlock_signals,
        }

    def _extract_opening_features(self, games):
        """开局特征：空列数、明牌分布、序列长度"""
        features = []
        for g in games:
            if g.steps:
                first_state = g.steps[0]
                features.append({
                    "empty_cols": first_state.empty_cols,
                    "face_up": first_state.face_up_count,
                    "avg_seq": first_state.avg_sequence_length,
                })
        return self._summarize(features)

    def _extract_deadlock_signals(self, games):
        """死局前 10 步的特征"""
        signals = []
        for g in games:
            if len(g.steps) >= 10:
                pre_deadlock = g.steps[-10:]
                signals.append({
                    "avg_empty_cols": mean(s.empty_cols for s in pre_deadlock),
                    "stock_remaining": pre_deadlock[-1].stock_remaining,
                    "completed": pre_deadlock[-1].completed,
                })
        return self._summarize(signals)
```

### 预期收益

- 发现人工难以观察的模式
- 指导策略优化方向
- 预测死局概率，提前调整策略
