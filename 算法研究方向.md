## 一、已实现方向的深化（高价值）

### 1.1 AlphaZero 自弈循环 ⭐⭐⭐⭐⭐

**现状**: 组件全部就绪（`NeuralStrategy` + `SelfPlayCollector` + `CurriculumScheduler` + `IterationEngine`），但没有端到端的训练管线。

**价值评估**: **极高** — 这是将现有零散组件串联为完整 AI 系统的关键路径。一旦实现，项目从"策略工具箱"升级为"能自我进化的 AI"。

**实现路径**:

```
AlphaZero 训练循环:
  自博弈(SelfPlayCollector) → 数据存储(Replay Buffer)
  → 模型训练(NeuralTrainer) → 模型评估(Evaluator) → 循环
```

**关键设计决策**:

| 决策点 | 选项 A | 选项 B | 建议 |
|--------|--------|--------|------|
| 训练数据 | 纯自弈 | 自弈 + 人类棋谱 | A（无棋谱可用） |
| 网络架构 | SimpleMLP（已有） | ResNet + Attention | A（先跑通再升级） |
| MCTS 集成 | 替换 rollout | 策略引导搜索 | B（与 PUCT 衔接） |
| 课程难度 | 固定 1 花色 | 渐进 1→2→4 | B（用 CurriculumScheduler） |

**预期收益**:
- 1 花色胜率: 0% → 40-60%（50 轮迭代后）
- 决策质量: 从人工启发式到自我进化
- 研究价值: 完整的 AlphaZero 在不完全信息卡牌游戏上的实践案例

**工作量**: 5-7 天

**现状**: `ISMCTSStrategy` 已实现基础框架，`determinization.py` 提供了 7 个采样函数。

**价值评估**: **高** — Spiderette 是不完全信息游戏（牌堆未知），IS-MCTS 理论上比标准 MCTS 更合理。

**深化方向**:

| 子方向 | 说明 | 优先级 |

| IS-PUCT 协同 | 确定化采样 + PUCT 策略引导搜索 | 高 |
| OBS (Optimized Best Response) | 计算对对手最优响应策略 | 低 |

**预期收益**: 2 花色胜率提升 5-15%

**工作量**: 3-4 天

## 三、新探索方向（高价值研究）

### 3.1 反事实遗憾最小化 (CFR) ⭐⭐⭐⭐⭐

**为什么值得探索**: Spiderette 是两人零和不完全信息博弈，CFR 是这类问题的理论最优解法之一。通过迭代"最小化遗憾"来收敛到纳什均衡策略。

**算法核心**:

```
对每个信息集 I:
  遗憾(I, a) = 实际回报(a) - 期望回报(当前策略)

策略更新:
  p(I, a) = max(遗憾(I, a), 0) / Σ max(遗憾(I', a'), 0)

收敛: 迭代 → 纳什均衡策略
```

**与现有代码的衔接**:

| 现有模块 | CFR 中的角色 |
|---------|-------------|
| `info_set.py` | 信息集抽象，直接复用 |
| `determinization.py` | 确定化采样，用于评估策略 |
| `rules.py` | 规则引擎，提供合法移动 |
| `types.py` | GameState/Move 类型 |

**实现路径**:

```
阶段 1: Vanilla CFR（2天）
  - 实现 CFRTrainer 类
  - 在 1 花色上验证收敛性
  - 对比 MCTS 胜率

阶段 2: CFR+ / Linear CFR（1天）
  - 加权遗憾累积，加速收敛
  - 实现剪枝（pruning）减少计算量

阶段 3: 与 MCTS 结合（2天）
  - CFR 输出作为 MCTS 的先验策略
  - 或 MCTS 作为 CFR 的评估器
```

**预期收益**:
- 理论上可以找到纳什均衡策略
- 4 花色胜率可能突破 5-10%（当前 0%）
- 学术价值：CFR 在非扑克类不完全信息博弈上的实践

**风险**: 状态空间大，收敛可能很慢。需要实验验证。

**工作量**: 4-5 天

---

### 3.2 蒙特卡洛遗憾最小化 (MCCFR) ⭐⭐⭐⭐

**为什么值得探索**: 标准 CFR 需要遍历所有信息集，计算量巨大。MCCFR 只采样部分信息集，大幅减少计算量。

**变体选择**:

```
CFR:     遍历所有信息集 → 精确但慢
MCCFR:   采样信息集 → 近似但快
  ├── Outcome Sampling (OS-MCCFR)
  ├── External Sampling (ES-MCCFR)  ← 最适合 Spiderette
  └── Random Walk Sampling
```

**建议**: 如果 CFR 收敛太慢，直接跳到 MCCFR。External Sampling MCCFR 最适合 Spiderette（每步只采样一个对手动作）。

**工作量**: 3-4 天（含实验验证）

---

### 3.3 策略蒸馏 (Policy Distillation) ⭐⭐⭐⭐

**为什么值得探索**: 强策略（MCTS/CFR）计算慢，弱策略（greedy）计算快但效果差。策略蒸馏可以将强策略的"知识"压缩到轻量网络中。

**实现思路**:

```
阶段 1: 用 MCTS/CFR 生成大量 (state, best_move) 数据
阶段 2: 训练一个小 MLP 模仿强策略的输出
阶段 3: 部署蒸馏后的网络作为独立策略

效果:
  MCTS (1000次迭代, 500ms/步) → 蒸馏网络 (1层, 5ms/步)
  胜率保留 80-90%
```

**与现有代码的衔接**:

| 现有模块 | 蒸馏中的角色 |
|---------|-------------|
| `NeuralTrainer` | 训练蒸馏网络 |
| `SelfPlayCollector` | 收集强策略的决策数据 |
| `MCTSStrategy` / `ISMCTSStrategy` | 教师策略 |
| `SimpleMLP` | 学生网络架构 |

**预期收益**:
- 推理速度提升 100 倍
- 胜率保留 80-90%
- 可以在前端实时使用强策略

**工作量**: 3 天

### 3.4 奖励塑形 (Reward Shaping) ⭐⭐⭐⭐

**为什么值得探索**: 当前 RL 环境的奖励信号极其稀疏（只有游戏结束时的 +1/-1）。奖励塑形可以提供中间反馈，加速学习。

**Spiderette 的奖励塑形设计**:

```python
def shaped_reward(state, move, next_state, outcome):
    reward = 0.0

    # 稀疏奖励（原始）
    if outcome == Outcome.WIN:
        reward += 10.0
    elif outcome == Outcome.DEADLOCK:
        reward -= 10.0

    # 密集奖励（塑形）
    reward += (next_state.completed - state.completed) * 2.0    # 完成牌堆
    reward += (next_state.face_up_count - state.face_up_count) * 0.1  # 翻牌
    reward += (next_state.empty_cols - state.empty_cols) * 0.5  # 空列
    reward += _suit_consistency_bonus(next_state) * 0.3         # 花色一致性

    if _is_near_deadlock(next_state):
        reward -= 5.0  # 死局惩罚

    return reward
```

**关键问题**: 奖励塑形必须满足"势函数"条件（potential-based shaping），否则会改变最优策略。需要数学验证。

**预期收益**: RL 训练收敛速度提升 3-5 倍

**工作量**: 2 天（设计 + 实现 + 验证）

### 3.5 元学习 (Meta-Learning) ⭐⭐⭐

**为什么值得探索**: 不同难度的最优策略差异很大。元学习可以让模型"学会如何快速适应新难度"。

**算法选择**: MAML 或 Reptile（一阶近似，更简单）

```
内循环: 在当前难度上快速适应（5 步梯度更新）
外循环: 更新初始参数，使适应能力最大化

效果:
  新难度只需 5-10 局游戏就能适应
  而非从头训练数千局
```

**风险**: MAML 需要二阶导数，实现复杂度高。可以先用 Reptile 验证可行性。

**工作量**: 4-5 天

---

### 3.6 集成策略 (Ensemble Strategy) ⭐⭐⭐

**为什么值得探索**: 不同策略在不同局面下各有所长。集成多个策略可以取长补短。

**实现方案**:

```python
class EnsembleStrategy:
    def __init__(self, strategies, method="voting"):
        self.strategies = strategies  # [greedy, mcts, neural, ...]
        self.method = method

    def choose(self, state, rules):
        if self.method == "voting":
            votes = defaultdict(int)
            for s in self.strategies:
                move = s.choose(state, rules)
                if move:
                    votes[move] += 1
            return max(votes, key=votes.get)
        elif self.method == "stacking":
            features = [self._strategy_preference(s, state, rules)
                        for s in self.strategies]
            return self.meta_learner.predict(features)
```

**预期收益**: 胜率提升 3-8%（通过多样性增益）

**工作量**: 2 天

### 3.7 对抗性鲁棒性 (Adversarial Robustness) ⭐⭐

**为什么值得探索**: 如果对手（发牌器）是"恶意"的，策略是否还能工作？与"死局检测"直接相关。

**研究问题**:
- 最坏情况发牌下，greedy 策略的胜率是多少？
- MCTS 能否找到对抗恶意发牌的鲁棒策略？

**价值**: 理论意义大于实际意义，但对理解策略极限很有帮助。

**工作量**: 3 天
